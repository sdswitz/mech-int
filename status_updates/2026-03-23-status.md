# Project Status: Mechanistic Interpretability with Sparse Autoencoders

**Date:** 2026-03-23
**Model:** Pythia-70M (EleutherAI), layer 3 residual stream (512-dim)
**Dataset:** OpenWebText, 50K texts (~5.88M tokens)

---

## What this project does

Trains sparse autoencoders (SAEs) on the residual stream of a small language model to decompose its internal representations into interpretable features, then analyzes those features through activation inspection, decoder geometry, and causal ablation.

---

## Completed work

### 1. Activation collection
- Ran Pythia-70M on 50K OpenWebText texts (truncated to 512 chars each)
- Cached layer 3 residual stream activations: 5,884,867 vectors of dim 512
- Checkpointing every 5K texts during collection

### 2. SAE training
- Trained SAEs at **4x, 8x, 16x, and 32x** expansion factors (all saved)
- Architecture: encoder (512 -> m) with ReLU, decoder (m -> 512) with unit-norm constraint on W_dec rows
- Training improvements added over time:
  - Cosine LR schedule with 2K-step linear warmup
  - Lambda warmup (ramps L1 penalty from 0 to target over 5K steps)
  - Gradient clipping (max norm 1.0)
  - `torch.randint` batch sampling (replaced `torch.randperm` which was slow on large datasets)
  - `mmap=True` for loading activations (needed for nodes with limited CPU RAM, e.g. 8GB)
  - Diagnostic print statements with `flush=True` at every stage for debugging kernel crashes on cloud compute
- 8x results (the most-inspected model): L0 ~ 0.05, MSE ~ 0.02, 4 dead features out of 4096

### 3. Feature inspection (`inspect-features.ipynb`)
- Top-K activation tracking (top 20 tokens per feature, memory-efficient)
- `inspect_feature(idx, k)` — shows top-k activating tokens with context
- `search_token(query)` — reverse lookup: find which features fire on a given token string
- Interpretability scoring: ranks features by consistency of their top-K tokens
- Dead feature detection
- Notable features found (8x model):
  - Feature 3440: "Reuters" detector (100% consistent)
  - Feature 2: "Feb" detector
  - Feature 2000: " un" prefix detector
  - Feature 3244: " re" prefix detector
- Notable features found (4x model):
  - Feature 1896: place-name detector (Angeles, Georgia, York, Vegas, Paris, Arizona)
  - Feature 2: " keep" detector
  - Feature 100: " northern" detector

### 4. Decoder cosine similarity analysis (`inspect-features.ipynb`)
- Pairwise cosine similarity of all W_dec rows
- 8x model: max off-diagonal similarity 0.80, mean 0.0015
- 4x model: max off-diagonal similarity 0.90, mean 0.0028
- Similarity distribution histogram
- High-similarity pair inspection (filtered to skip single-char token features)
- Hierarchical clustering with Ward linkage + dendrogram visualization

### 5. Causal ablation pipeline (`ablation.ipynb`)
- Ablation method: encode residual stream through SAE, compute `f_i * W_dec[i]`, subtract from residual stream via TransformerLens hook at `blocks.3.hook_resid_post`
- Metrics: per-token KL divergence, top-5 prediction comparison, greedy generation comparison
- Tested on:
  - Feature 3440 (Reuters, 8x) with Reuters-style text: **zero KL** — pure recognizer, not causally important
  - Feature 1896 (place names, 4x) with "Eiffel Tower in Paris" text: **small but nonzero KL** (max 0.05 at "Paris"), subtle distribution shifts (e.g. country name reranking) but no change in greedy output

## Current status and next steps

The single-feature ablation pipeline works but hasn't yet found features with large causal effects. The main next step is **batch ablation** — ablating each of the m features one at a time across a diverse eval set to rank all features by mean KL divergence. This would identify which features actually drive model behavior vs. which are passive recognizers.

Other next steps:
- Cross-expansion feature comparison (match features across 4x/8x/16x/32x via decoder cosine similarity, look for feature splitting)
- Multi-feature ablation (remove groups of related features simultaneously)
- Longer training runs if quality improves (currently 50K epochs)

## Practical notes for running on cloud compute

- **CPU RAM matters:** The activation file is ~12GB. Nodes with <16GB RAM will OOM during `torch.load()`. The fix is `torch.load(..., mmap=True, weights_only=True, map_location='cuda')`.
- **Batch sampling:** Use `torch.randint` not `torch.randperm` — permuting 5.8M indices every step is extremely slow.
- **Path verification:** Run `python check_paths.py` after cloning to verify all directories and files are in place.
- **Training a new expansion:** `python train.py --expansion 16 --lam 3` — auto-skips if model already exists.
- **Inspecting a different expansion:** Change `EXPANSION = 8` at the top of `inspect-features.ipynb` or `ablation.ipynb`.
