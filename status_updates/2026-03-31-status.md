# Project Status: Default Research Pipeline Refactor

**Date:** 2026-03-31  
**Scope:** Manual SAE research workflow for Pythia-70M / layer 3 residual stream  
**Goal of this update:** Record the pipeline refactor that makes the default research workflow modular, script-first, and reproducible before further model experimentation or autoresearch work.

---

## What changed

The repository is no longer centered on a single monolithic training script plus analysis notebooks. The default manual workflow now has three layers:

1. **Shared package code** in `mechint/`
2. **CLI entrypoints** in `scripts/`
3. **Run artifacts** under `runs/`

The research target itself did **not** change:
- Model: Pythia-70M
- Hook point: `blocks.3.hook_resid_post`
- Activation dim: 512
- Dataset/cache target: `activations/activations_50000.pt`

This refactor was about repo structure and reproducibility, not changing the scientific setup.

---

## New default pipeline

### Shared package: `mechint/`

Core logic was extracted into reusable modules:

- `mechint/config.py` — JSON-backed SAE training config dataclass
- `mechint/data.py` — device detection, deterministic train/val split, streamed batch loading, run-dir helpers, activation collection helpers
- `mechint/sae.py` — canonical `SparseAutoEncoder` and `SAEloss`
- `mechint/eval.py` — post-training evaluation, metrics summaries, metrics/summary writing
- `mechint/analysis.py` — top activations, token search helpers, decoder cosine similarity helpers
- `mechint/ablation.py` — single-feature ablation hooks, batch feature ranking by KL, checkpoint loading

This removes duplication between training code, notebooks, and future automation.

### Script entrypoints: `scripts/`

The default research workflow is now script-first:

- `scripts/collect_activations.py`
- `scripts/train_sae.py`
- `scripts/eval_sae.py`
- `scripts/batch_ablate.py`
- `scripts/compare_expansions.py`

These are the intended primary interfaces for future manual work.

### Run artifact layout: `runs/`

New experiments are designed to write outputs into a per-run directory under `runs/`, including:

- `config.json`
- `metrics.csv`
- `checkpoint.pt`
- `summary.json`
- `manifest.json`

Legacy locations (`saved_models/`, `training_metrics/`) are still supported for compatibility, but they are no longer the intended long-term layout.

---

## Backward compatibility preserved

To avoid breaking the old workflow immediately:

- `sae.py` now re-exports the canonical implementation from `mechint.sae`
- `train.py` now forwards to `scripts/train_sae.py`
- the top-level `train.py` wrapper defaults to a legacy-layout mode so old habits still produce `saved_models/sae_model_{expansion}x.pt` and `training_metrics/training_metrics_{expansion}x.csv`

This means existing code and imports should continue to work while new work migrates to the refactored pipeline.

## What is still unfinished

The refactor is not fully complete in one important sense: the canonical notebooks are still notebook-native.

Still pending:

- migrate `inspect-features.ipynb` to import the new analysis helpers instead of keeping inline implementations
- migrate `ablation.ipynb` to import the new ablation helpers instead of defining them in cells
- decide whether to archive or keep additional legacy files such as `train_wandb.py` and `colab-training.ipynb`
- optionally add richer eval metrics and run manifests as the research workflow stabilizes

So the pipeline is now solid enough to build on, but the notebook layer has not yet been fully brought into the same modular structure.

---

## Recommended next step

Before doing more architectural changes or starting autoresearch work, the highest-value cleanup step is:

**Convert the remaining canonical notebooks into thin consumers of `mechint.analysis` and `mechint.ablation`.**

That would complete the transition from a notebook-first repo to a true script-and-library research pipeline while preserving the exploratory notebook workflow.
